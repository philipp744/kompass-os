# Statusmodell

Das Statusmodell verhindert, dass Ideen, Annahmen und Entscheidungen vermischt werden.

## Objektstatus

| Status | Bedeutung |
|---|---|
| Idee | Noch nicht geprüft |
| Hypothese | Plausible Annahme, aber nicht validiert |
| Research | Erste Daten oder Quellen vorhanden |
| Kandidat | Potenziell relevant, weitere Prüfung nötig |
| Priorität A | Hohe Priorität |
| Priorität B | Mittlere Priorität |
| Pausiert | Nicht aktiv, aber nicht verworfen |
| Validiert | Durch Daten oder Test bestätigt |
| Verworfen | Nicht weiterverfolgen, Begründung dokumentiert |

## Informationsklassen

| Klasse | Bedeutung |
|---|---|
| Fakt | Objektiv belegbare Information |
| Beobachtung | Aus Research oder Feedback abgeleitet |
| Hypothese | Noch zu prüfende Annahme |
| Entscheidung | Bewusst getroffen und begründet |

## Pflichtregel

Jede Änderung von Status `Priorität A`, `Validiert` oder `Verworfen` benötigt eine Begründung.
