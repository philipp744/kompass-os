# Workflow: Readability Review

## Purpose
Ensure Kompass OS remains usable for Philipp and not only for technical agents.

## When to run
Run after every major sprint or structural change.

## Checklist

### Entry points
- README has a clear start path
- PROJECT_STATUS is current
- docs/navigation is current

### Human language
- important files explain why they exist
- acronyms are in docs/glossary
- no critical decision is hidden only in code

### Dashboard readiness
- new entities have labels and statuses
- new workflows have clear next actions
- new prototypes have README files

### Business clarity
- current priority is visible
- next decision gate is visible
- risks are visible
- done vs not done is visible

## Output
The review should create or update:
- PROJECT_STATUS.md
- docs/navigation.md
- docs/glossary.md
- relevant pilot README or decision map

## Rule
If a file is technically correct but not understandable, it is not finished.
