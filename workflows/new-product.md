# Workflow: Neue Produktidee

Dieser Workflow verhindert, dass Produkte ohne Problembezug bewertet werden.

## Reihenfolge

1. Produktidee erfassen
2. Zugehöriges Problem benennen
3. Zielgruppe benennen
4. Einsatzsituation beschreiben
5. Wettbewerber oder Substitute suchen
6. Differenzierung prüfen
7. Lieferantenannahme prüfen
8. Bundle-Fähigkeit prüfen
9. Temu-/Amazon-Austauschbarkeit prüfen
10. Status setzen

## Pflichtfragen

- Welches konkrete Problem löst das Produkt?
- Für wen ist das Problem relevant?
- Wie häufig tritt das Problem auf?
- Warum wäre ein eigener Shop/Marke plausibel?
- Warum ist das Produkt nicht nur ein austauschbarer Marketplace-Artikel?
- Welche Landingpage-Hypothese kann damit getestet werden?

- Priorität B
- Research nötig
- Pausiert
