# Entitätenmodell

Kompass OS modelliert Wissen als verknüpfte Objekte.

## Kernentitäten

| Präfix | Entität | Beschreibung |
|---|---|---|
| M | Markt | Übergeordneter Markt oder Themenbereich |
| ZG | Zielgruppe | Klar unterscheidbare Käufer- oder Nutzergruppe |
| PRO | Problem | Kaufrelevanter Schmerzpunkt |
| SIT | Situation | Kontext, in dem ein Problem auftritt |
| BC | Business Case | Bündel aus Markt, Zielgruppe, Problem und Lösungsansatz |
| PF | Produktfamilie | Gruppe verwandter Produkte |
| PRD | Produkt | Konkretes Produkt oder Produktkonzept |
| KW | Keyword | Suchbegriff oder Suchcluster |
| LP | Landingpage | Validierungs- oder Verkaufsseite |
| SEO | Content | Ratgeber, Blogartikel, Kategorie-Content oder Vergleich |
| LIF | Lieferant | Lieferant, Hersteller oder Beschaffungsquelle |
| WET | Wettbewerber | Marke, Shop, Marketplace-Angebot oder Alternative |
| SRC | Quelle | Externe oder interne Quelle |
| HYP | Hypothese | Zu prüfende Annahme |
| DEC | Entscheidung | Bewusste Entscheidung mit Begründung |
| EXP | Experiment | Validierungstest, zum Beispiel Landingpage oder Ads |
| TASK | Aufgabe | Operativer nächster Schritt |

## Grundregel

Keine wichtige Beziehung darf nur im Freitext stehen. Wenn sie später abgefragt werden soll, braucht sie eine eindeutige ID-Verknüpfung.
