# GUI Roadmap

## Vision
Kompass OS will evolve from documents and spreadsheets into a web-based operating system.

## Guiding principle
The GUI is a view on the data, not the source of truth.

Source systems:
- Google Sheets: operational data entry
- GitHub: versioned knowledge and implementation
- Future database: scalable storage if required

## Planned user interfaces

### 1. Project Dashboard
Purpose:
- project progress
- roadmap
- open work
- architecture status
- releases

### 2. Business Dashboard
Purpose:
- markets
- business cases
- scores
- landing pages
- SEO
- experiments
- suppliers
- products

### 3. Business Case Workspace
One screen containing:
- target groups
- situations
- problems
- products
- validation
- decisions
- timeline

## Design principles
- mobile friendly
- dashboard first
- drill-down navigation
- minimal manual input
- visual status indicators

## Not for v1
- no ERP complexity
- no custom database before needed
- no duplicated data

## Long-term objective
A user should be able to manage hundreds of business cases from the GUI while GitHub and Sheets remain the underlying sources of truth.
